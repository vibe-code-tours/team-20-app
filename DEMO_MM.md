# Demo Script — Team 20 (Myanmar)

## ၆ မိနစ် Demo Narration

```text
0:00–0:35 — စတင်မိတ်ဆက်
မင်္ဂလာပါ။ ကျွန်တော်တို့က Team 20 ဖြစ်ပါတယ်။ ကျွန်တော်တို့ တည်ဆောက်ထားတာက Kone Sone Sine ဆိုတဲ့ fundraising website ပါ။ ဒီ website က customer တွေ နဲ့ organizer တွေအတွက် preorder စနစ်ကို ပိုလွယ်ကူ၊ ပိုမြန်၊ ပိုရှင်းအောင် လုပ်ပေးပါတယ်။ အရင်က fundraising event တွေမှာ chat message တွေ၊ spreadsheet တွေ၊ manual follow-up တွေနဲ့ပဲ လုပ်ရတာများပါတယ်။ အဲဒါကြောင့် order လွဲတာ၊ payment စစ်တာ နှေးတာ၊ အလုပ်ပိုလာတာတွေ ဖြစ်တတ်ပါတယ်။ ကျွန်တော်တို့ app က အဲဒီအလုပ်တွေကို တစ်နေရာတည်းမှာ စုစည်းပေးထားပါတယ်။

0:35–1:00 — ကျွန်တော်တို့ ဘာတွေ တည်ဆောက်ထားလဲ
ဒီ system မှာ role သုံးမျိုး ပါပါတယ် — customer, organizer, admin ပါ။ Customer က event တွေကြည့်ပြီး order လုပ်နိုင်ပါတယ်။ Organizer က order တွေ manage လုပ်နိုင်တယ်၊ analytics ကြည့်နိုင်တယ်၊ data export လုပ်နိုင်တယ်။ Admin က organizer အသစ်တွေကို invite လုပ်ပြီး access ကို ထိန်းချုပ်နိုင်ပါတယ်။

1:00–2:45 — Customer flow
အခု customer flow ကို စပြပါမယ်။ Home page မှာ fundraising brand ကို အရင်မြင်ရပြီး upcoming events တွေကိုလည်း တစ်ခါတည်းကြည့်နိုင်ပါတယ်။ Browse events ကိုနှိပ်ပြီး event detail page ထဲဝင်ရင် event date, location, preorder deadline တွေကို ကြည့်နိုင်ပါတယ်။

ပြီးတော့ menu page ထဲဝင်ပြီး order စတင်လုပ်ပါတယ်။ ဒီနေရာမှာ items တွေကို category အလိုက်မြင်ရတယ်။ လိုတဲ့ item ကို cart ထဲထည့်နိုင်ပြီး right side မှာ order summary ကိုလည်း တစ်ခါတည်းမြင်နိုင်ပါတယ်။ Checkout form မှာ name, phone number, note တွေ ထည့်ပြီး order တင်နိုင်ပါတယ်။ အဲဒါကြောင့် order ကို နောက်မှ လွယ်လွယ်ကူကူ ရှာနိုင်ပါတယ်။

Order တင်ပြီးနောက် payment screenshot ကို upload လုပ်ပါတယ်။ Upload လုပ်ပြီးသွားရင် payment success screen ပြပြီး order number လည်း ထွက်လာပါတယ်။ အဲဒီ order number နဲ့ later မှာ Track Order page ထဲကနေ order status ကို စစ်နိုင်ပါတယ်။ Customer က current status, event details, ordered items, contact information တွေကို တစ်နေရာတည်းမှာ ကြည့်နိုင်ပါတယ်။ ဒါကြောင့် order လက်ခံပြီးမပြီး စိုးရိမ်နေစရာ မလိုတော့ပါဘူး။

2:45–4:05 — Organizer နဲ့ Admin flow
အခု organizer side ကို ပြပါမယ်။ Organizer account နဲ့ login ဝင်ပြီးရင် dashboard overview ကို တွေ့ရပါတယ်။ အဲဒီမှာ total orders, revenue, pending orders, completed orders စတဲ့အချက်အလက်တွေကို ချက်ချင်းမြင်ရပါတယ်။ Sidebar ကနေ events, orders, menu, analytics, export ဆိုပြီး လိုတဲ့ section ကို တန်းသွားနိုင်ပါတယ်။

ဒီ dashboard က organizer တွေအတွက် decision လုပ်ရလွယ်အောင် ပြထားတာပါ။ Message တွေ၊ spreadsheet တွေကို လိုက်ရှာနေစရာ မလိုဘဲ ဘာလုပ်ရမယ်၊ ဘာပြီးပြီလဲ၊ event က ဘယ်လိုအခြေအနေရှိလဲ ဆိုတာကို တစ်ချက်ကြည့်နဲ့ သိနိုင်ပါတယ်။ လိုအပ်ရင် data export လည်း လုပ်နိုင်ပါတယ်။

Admin မှာတော့ login နဲ့ dashboard ကို သီးခြားထားထားပါတယ်။ ဒါကြောင့် access control ပိုရှင်းပြီး လုံခြုံပါတယ်။ Admin side မှာ invitations နဲ့ users ကို စီမံနိုင်ပါတယ်။

4:05–5:10 — Organizer invitation နဲ့ registration
ဒီ flow က အရမ်းအရေးကြီးပါတယ်။ Admin က organizer အသစ်အတွက် invitation တစ်ခု create လုပ်နိုင်ပါတယ်။ အဲဒီအခါ secure invitation code နဲ့ share လုပ်လို့ရတဲ့ registration link တစ်ခု ထွက်လာပါတယ်။ ဒါကြောင့် password ကို လက်ဆင့်ကမ်းပေးစရာ မလိုတော့ဘဲ onboarding process က ပိုလွယ်သွားပါတယ်။

Invite လက်ခံမယ့် organizer က link ကိုဖွင့်ပြီး invitation code ထည့်မယ်၊ နာမည်၊ email, password တွေ ဖြည့်မယ်၊ ပြီးရင် account create လုပ်မယ်။ Registration အောင်မြင်ပြီးတာနဲ့ သူ့ role နဲ့တကွ organizer dashboard ထဲကို တန်းဝင်နိုင်ပါတယ်။ အဲဒါကြောင့် admin က access ကို ထိန်းထားနိုင်ပြီး organizer ကလည်း ချက်ချင်းအလုပ်စနိုင်ပါတယ်။

5:10–5:45 — Tech highlight
ဒီ project မှာ အခက်ဆုံးအပိုင်းက role-based flow တွေနဲ့ order lifecycle ကို app တစ်ခုလုံးမှာ တသမတ်တည်းဖြစ်အောင် ထိန်းရတာပါ။ OpenAI API ကို chatbot နဲ့ order extraction အတွက် သုံးထားပါတယ်။ Payment screenshot upload feature လည်း ထည့်ထားလို့ offline payment ကိုလည်း organize လုပ်ရလွယ်ပါတယ်။ Order data နဲ့ stock state တွေကိုလည်း တသမတ်တည်းဖြစ်အောင် စီမံထားပါတယ်။

5:45–6:00 — နောက်ဆက်တွဲ နဲ့ အပြီးသတ်
နောက်ထပ်အဆင့်အနေနဲ့ payment verification ကို ပိုကောင်းအောင်လုပ်မယ်၊ order update notification တွေ ထည့်မယ်၊ organizer အတွက် analytics ကိုလည်း ပိုချဲ့မယ်။ ကြည့်ပေးတဲ့အတွက် ကျေးဇူးတင်ပါတယ်။ Team 20 ပါ။
```
